## 1️⃣ ESPHome реализация

### 📁 Структура файлов

```
ESPHome/
├── LD2450_with_native_component.yaml     # ⭐ Конфигурация ESPHome
├── Plotly_Graph_Native_Component.txt     # Карточка для HA
├── README.md                              # Документация
└── secrets.yaml.example                   # Шаблон секретов
```

### 🎯 Какую конфигурацию выбрать?

#### ⭐ `LD2450_with_native_component.yaml` - РЕКОМЕНДУЕТСЯ

**Особенности:**
- ✅ Нативный компонент ESPHome для LD2450
- ✅ 3 аппаратные зоны радара (на уровне LD2450)
- ✅ Автоматическое включение режима 3 целей
- ✅ Переключатель в Home Assistant
- ✅ Все сенсоры для 3 целей (X, Y, Speed, Angle)
- ✅ Веб-интерфейс для настройки
- ✅ Минималистичная конфигурация (~850 строк)

**Когда использовать:**
- Новые проекты
- Нужно 3 зоны или меньше
- Важна производительность
- Нужен веб-интерфейс

### ⚡ Почему ESP-IDF Framework?

**Путь:** `ESPHome/LD2450_with_native_component.yaml`

**Преимущества ESP-IDF:**
- ✅ Стабильнее UART на 256000 baud
- ✅ Меньше размер прошивки (~1.0 MB, -17%)
- ✅ Меньше потребление RAM (~130 KB, -13%)
- ✅ Лучшая производительность (+10-15%)
- ✅ Будущее ESPHome (переход на ESP-IDF по умолчанию)

**Рекомендуется использовать ESP-IDF для всех новых проектов!**

### 🚀 Быстрый старт ESPHome

#### 1. Настройка секретов

```bash
cd ESPHome/
cp secrets.yaml.example secrets.yaml
nano secrets.yaml
```

Заполните:
```yaml
wifi_ssid: "YOUR_WIFI_SSID"
wifi_password: "YOUR_WIFI_PASSWORD"
ota_password: "YOUR_OTA_PASSWORD"
hotspot_password: "YOUR_HOTSPOT_PASSWORD"

# Генерация ключа: esphome encryption-key
api_encryption_key: "YOUR_API_ENCRYPTION_KEY"
```

#### 2. Компиляция и прошивка

```bash
cd ESPHome/
esphome compile LD2450_with_native_component.yaml
esphome upload LD2450_with_native_component.yaml
```

#### 3. Проверка работы

```bash
esphome logs LD2450_with_native_component.yaml
```

**В логах должно быть:**
- ✅ `Target count: 3`
- ✅ `Multi-Target mode: ON`
- ✅ Данные от радара (X, Y координаты)

#### 4. Добавление в Home Assistant

1. Откройте **Settings → Devices & Services**
2. Найдите **"ld2450"** в обнаруженных устройствах
3. Нажмите **"Configure"**
4. Введите ваш `api_encryption_key` из `secrets.yaml`
5. Готово! 🎉

### 🔧 Ключевые компоненты конфигурации

#### UART настройка (256000 baud - ОБЯЗАТЕЛЬНО!)
```yaml
uart:
  id: uart_bus
  tx_pin: GPIO17  # TX → RX радара
  rx_pin: GPIO16  # RX ← TX радара
  baud_rate: 256000
  parity: NONE
  stop_bits: 1
  data_bits: 8
```

#### Нативный компонент LD2450
```yaml
ld2450:
  id: ld2450_radar
  uart_id: uart_bus
```

#### Переключатель режима множественных целей
```yaml
switch:
  - platform: ld2450
    multi_target:
      name: "Режим нескольких целей"
      id: multi_target_mode
      restore_mode: RESTORE_DEFAULT_ON  # Автоматически ВКЛ
```

#### Аппаратные зоны радара (3 зоны)
```yaml
number:
  - platform: ld2450
    zone_1:
      x1: { name: "Зона 1 - X1" }
      y1: { name: "Зона 1 - Y1" }
      x2: { name: "Зона 1 - X2" }
      y2: { name: "Зона 1 - Y2" }
    presence_timeout: { name: "Зона 1 - Таймаут" }
  # ... аналогично для zone_2 и zone_3
```

**Дефолтные зоны:**
- **Зона 1:** 2м × 0.75м (глубина 250-1000мм)
- **Зона 2:** 4м × 2м (глубина 1000-3000мм)
- **Зона 3:** 6м × 3м (глубина 3000-6000мм)

## 2️⃣ Home Assistant карточки (HASSio)

### 📦 Plotly Graph Card

**Установка через HACS:**

1. Откройте **HACS** в Home Assistant
2. Перейдите в **Frontend**
3. Нажмите **⊕ Explore & Download Repositories**
4. Найдите **"Plotly Graph Card"**
5. Нажмите **Download**
6. Перезагрузите Home Assistant

**Ссылка на проект:**
🔗 https://github.com/dbuezas/lovelace-plotly-graph-card

### 📊 Добавление карточки

1. Откройте Dashboard → **Edit** → **Add Card**
2. Выберите **Manual** внизу
3. Скопируйте содержимое файла:

**Для нативного компонента:**
```
ESPHome/Plotly_Graph_Native_Component.txt
```

4. Вставьте в редактор
5. Если изменили `entity_name` в конфигурации, замените `ld2450` на своё
6. Сохраните

### 🎨 Что отображается на карточке

#### Цели (Person 1-3)
- 🔵 **Синие/Оранжевые/Зелёные точки**
- Отображают текущее положение людей
- Обновляются в реальном времени
- Координаты в миллиметрах

#### Аппаратные зоны (HW Zone 1-3)
- **Зона 1:** Зелёный прямоугольник (полупрозрачный)
- **Зона 2:** Оранжевый прямоугольник (полупрозрачный)
- **Зона 3:** Синий прямоугольник (полупрозрачный)
- Зоны отображаются как прямоугольники с координатами из Home Assistant

#### Область покрытия (Coverage)
- Серая пунктирная линия
- Форма "груши" - максимальная дальность ~6м по центру
- Примерная зона обнаружения датчика

### ⚙️ Настройка карточки

**Основные параметры в `Plotly_Graph_Native_Component.txt`:**

```yaml
type: custom:plotly-graph
title: "LD2450 - Native Component"
refresh_interval: 2  # Обновление каждые 2 секунды
hours_to_show: current_day
```

**Изменение refresh_interval:**
- Для экономии ресурсов: увеличьте до 5-10 секунд
- Для быстрого обновления: уменьшите до 1 секунды

**Изменение масштаба:**
```yaml
xaxis:
  range: [-4000, 4000]  # -4м до +4м
yaxis:
  range: [8000, 0]      # 0м до 8м
```

### 📐 Структура карточки

**Цели:**
```yaml
entities:
  - entity: ''

      - $ex hass.states["sensor.ld2450_tsel_1_y"].state
```

**Зоны:**
```yaml

      - $ex hass.states["number.ld2450_zona_1_x2"].state*1
      # ... и так далее для прямоугольника
```

## 3️⃣ Предварительная настройка радара через USB-UART

### 🔌 Подключение радара к компьютеру

#### Через USB-UART переходник (CH340, CP2102, FT232)

**Схема подключения:**
```
LD2450                    USB-UART переходник
─────────────────         ───────────────────
VCC (5V)  ────────────►   5V
GND       ────────────►   GND
TX        ────────────►   RX (порт компьютера)
RX        ────────────►   TX (порт компьютера)
```

**Важно:**
- ⚠️ **НЕ ПОДКЛЮЧАЙТЕ VCC** если радар уже питается от ESP32!
- ⚠️ Радар требует **5V 500mA минимум**
- ⚠️ UART работает на **3.3V логике**, но питание 5V

### 📡 Скрипты для работы с радаром

#### 1. Подключение радара в WSL (если используете WSL2)

```bash
# Скрипт автоматического подключения
./scripts/connect_radar.sh
```

**Что делает:**
- Автоматически привязывает USB устройство через usbipd
- Подключает радар к WSL
- Проверяет доступность порта `/dev/ttyUSB0`

**Требования:**
- Windows с установленным `usbipd-win`
- WSL2
- Радар подключен к USB порту Windows (например, COM10)

#### 2. Чтение данных радара

**Python скрипт для мониторинга:**
```bash
python3 scripts/radar.py
```

**Что выводит:**
- Данные в реальном времени
- Координаты X, Y в миллиметрах
- Счётчик обработанных целей
- Правильный парсинг протокола LD2450 (AA FF)

**Требования:**
- Python 3 с библиотекой `pyserial`: `pip install pyserial`

#### 3. Настройка через официальный инструмент HLK

**Официальный инструмент HLK:**
- Windows приложение: `HLK-LD2450_TOOL_English.exe`
- Используйте официальный инструмент для настройки параметров радара

**⚠️ Внимание:** LD2450 использует **бинарный протокол**, а не AT команды!

### 🔧 Настройка через официальный инструмент

**Официальный инструмент HLK:**
- Windows приложение: `HLK-LD2450_TOOL_English.exe`
- Скачать: официальный сайт Hi-Link или репозиторий с документацией LD2450

**Как использовать:**
1. Подключите радар к COM порту Windows
2. Откройте `HLK-LD2450_TOOL_English.exe`
3. Выберите COM порт
4. Настройте параметры:
   - Зоны (Zone 1-3)
   - Чувствительность
   - Режим работы
   - Частота обновления
5. Нажмите "Write" для сохранения в радар

**Конфигурация сохраняется в радаре и будет загружена автоматически при подключении к ESP32!**

### 📋 Бинарный протокол LD2450

**Формат пакета данных (от радара):**
```
Заголовок: AA FF
Разделитель: 55 CC
Данные: 3 цели × 8 байт каждая
```

**Формат команды (к радару):**
```
Header: FD FC FB FA
Length: 2 байта (little-endian)
Command: 2 байта
Data: N байт
Footer: 04 03 02 01
```

**Примеры команд (из ESPHome конфигурации):**

**1. Установка чувствительности (0x64):**
```cpp
uint8_t cmd_sensitivity[] = {
  0xFD, 0xFC, 0xFB, 0xFA,  // Header
  0x0A, 0x00,              // Length = 10 bytes
  0x64, 0x00,              // Command: Set sensitivity
  sens, sens, sens, sens, sens, sens, sens, sens,  // 8 диапазонов
  0x04, 0x03, 0x02, 0x01   // Footer
};
```

**2. Установка порога энергии (0x60):**
```cpp
uint8_t cmd_energy[] = {
  0xFD, 0xFC, 0xFB, 0xFA,  // Header
  0x04, 0x00,              // Length = 4 bytes
  0x60, 0x00,              // Command: Set energy threshold
  (uint8_t)(energy & 0xFF),        // Low byte
  (uint8_t)((energy >> 8) & 0xFF), // High byte
  0x04, 0x03, 0x02, 0x01   // Footer
};
```

**3. Сохранение конфигурации (0x60 0x61):**
```cpp
uint8_t cmd_save[] = {
  0xFD, 0xFC, 0xFB, 0xFA,
  0x02, 0x00,
  0x60, 0x61,              // Command: Save config
  0x04, 0x03, 0x02, 0x01
};
```

### 🎛️ Настройка параметров через ESPHome

После подключения радара к ESP32, настройки можно изменять через Home Assistant:

**1. Откройте устройство в HA:**
- Settings → Devices → ld2450 → Configure

**2. Используйте кнопки:**
- **"Применить зоны по умолчанию"** - установит дефолтные зоны
- **"Записать конфигурацию в радар"** - сохранит настройки в радар
- **"Применить и перезагрузить радар"** - применит настройки и перезагрузит

**3. Настройте параметры:**
- **Чувствительность радара** (0-9)
- **Порог энергии** (100-10000)
- **Время удержания цели** (0-60 сек)
- **Фильтр скорости** (0-100 см/с)
- **Режим работы** (Normal/High Sensitivity/Low Power/High Precision)
- **Частота обновления** (1-20 Гц, макс 10 Гц для Multi-Target)

**4. Настройте аппаратные зоны:**
- Измените координаты зон (X1, Y1, X2, Y2)
- Установите таймаут присутствия для каждой зоны

**⚠️ Важно:** После изменения зон, нажмите **"Применить и перезагрузить радар"** для сохранения!

## ❓ Часто задаваемые вопросы

### ESPHome

**Q: Почему только ESP-IDF?**  
A: ESP-IDF обеспечивает лучшую производительность (+10-15%), стабильнее UART на 256000 baud, и является будущим ESPHome (переход на ESP-IDF по умолчанию).

**Q: Радар не появляется в Home Assistant**  
A: Проверьте `api_encryption_key` - должен совпадать в `secrets.yaml` и HA. Перезагрузите ESP32.

**Q: Как включить режим 3 целей?**  
A: Переключатель "Режим нескольких целей" включится автоматически. Или включите вручную в HA.

**Q: Радар перезагружается**  
A: Проверьте питание - нужны стабильные 5V 500mA. Используйте качественный USB кабель или внешний БП.

### Home Assistant

**Q: Карточка Plotly не отображается**  
A: Убедитесь что установили Plotly Graph Card через HACS и перезагрузили HA.

**Q: Цели не отображаются на карточке**  
A: Проверьте имена сущностей в карточке - они должны совпадать с именами в вашей конфигурации ESPHome.

**Q: Зоны не отображаются**  
A: Убедитесь что зоны настроены в ESPHome и их координаты доступны в HA.

### USB-UART

**Q: Не могу найти порт /dev/ttyUSB0**  
A: Если используете WSL, запустите `./scripts/connect_radar.sh`. Если Linux - проверьте `lsusb` и `dmesg`.

**Q: Данные не приходят от радара**  
A: Проверьте скорость UART - должно быть **256000 baud**. Проверьте подключение TX/RX (перекрёстное).

**Q: Как сохранить настройки в радаре?**  
A: Используйте кнопку "Записать конфигурацию в радар" в Home Assistant или официальный инструмент HLK.
